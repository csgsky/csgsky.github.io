代理模式

代理模式(Proxy Pattern) 也称作委托模式。在代理模式中，两个对象参与处理同一个请求，接受请求的对象将请求委托给另一个对象处理，代理模式使得我们可以使用聚合来代替继承。委托者的作用是将委托者与实际现实代码分离出来，以达成解耦的目的。

1.1 属性的委托

Kotlin 类的委托和 Java 的类的委托大同小异。唯一不同的就是生成代理类的时候，Java 会使用构造方法去委托类，而 Kotlin 直接通过 by 运算符约定了。而属性的委托算是 kotlin 最独特和最强大的功能之一。它可以让你轻松实现这样的属性，它们出来起来比把值存在支持字段中更复杂，却不用在每个访问器中都重复这样的逻辑。
委托属性赋予了属性一下几种能力：

- 延迟属性（lazy properties）: 其值只在首次访问时计算。
- 可观察属性（observable properties）: 监听器会收到此属性变更的通知。
- 把多个属性映射成（map）中，而不是每个单独存在每个字段中。

惰性初始化是一种常见的模式，直到第一次访问该属性的时候，才根据需要，创建对象的一部分。当初始化需要创建大量资源，并且使用对象时，并不是总需要数据，非常有用。

小结

1.委托属性可以用来重用逻辑，这些逻辑控制如何存储、初始化、访问和修改属性值，这是用来构建框架的强大工具。
